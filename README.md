# My Data Visualization Portfolio

Built with [Quarto](https://quarto.org) and hosted on GitHub Pages.

## Project structure

```
portfolio/
├── _quarto.yml          # Site config, navbar, theme
├── custom.scss          # Dark theme styles
├── index.qmd            # Homepage with hero
├── about.qmd            # Contact page
├── chart/
│   ├── index.qmd        # Charts gallery (auto-generated listing)
│   └── my-first-chart/
│       ├── index.qmd    # The chart page
│       └── thumbnail.png  # Card thumbnail (use ggsave or screenshot)
├── db/
│   ├── index.qmd        # Dashboards gallery
│   └── my-dashboard/
│       ├── index.qmd
│       └── thumbnail.png
└── blog/
    ├── index.qmd        # Stories gallery
    └── my-story/
        ├── index.qmd
        └── thumbnail.png
```

## Adding a new chart

1. Create a new folder: `chart/my-chart-name/`
2. Add `index.qmd` with this frontmatter:
   ```yaml
   ---
   title: "Chart Title"
   date: 2025-05-01
   categories: [R, TidyTuesday]
   image: thumbnail.png
   ---
   ```
3. Add your R/Python/D3 code below the frontmatter
4. Save a thumbnail: `ggsave("thumbnail.png", width=8, height=5)`
5. Run `quarto render` then push to GitHub

## Deploying to GitHub Pages

1. `quarto render` — builds the site into `docs/`
2. `git add . && git commit -m "update" && git push`
3. In GitHub repo Settings → Pages → Source: `main` branch, `/docs` folder

## Tips for a PNG visualization

If your viz is just a PNG (not generated by code):

```yaml
---
title: "My Chart"
date: 2025-05-01
categories: [R]
image: my-chart.png
---

![](my-chart.png)
```

Just drop the PNG in the folder alongside `index.qmd`.
